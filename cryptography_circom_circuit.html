What is Circom?

Circom is a domain-specific language (DSL) for writing arithmetic circuits used in zero-knowledge proofs. It compiles high-level circuit descriptions into the low-level constraints needed for zk-SNARK proof systems.

Zero-Knowledge Proof Properties

🔒
Completeness
If the statement is true, an honest prover can convince an honest verifier
🛡️
Soundness
If the statement is false, no cheating prover can convince an honest verifier
🕵️
Zero-Knowledge
The verifier learns nothing about the secret beyond the fact that the statement is true
⚡
Succinctness
Proofs are small and fast to verify, regardless of computation complexity

Writing Circom Circuits

Circom circuits define computational logic using templates, signals, and constraints:

pragma circom 2.0.0; // Template for proving knowledge of square root template SquareRoot() { // Private input (secret) signal private input x; // Public input (known value) signal input square; // Output signal signal output valid; // Constraint: x * x must equal square square === x * x; // Output 1 if constraint is satisfied valid <== 1; } // Main component instantiation component main = SquareRoot();

This circuit proves: "I know a secret number x such that x² = square" without revealing x.

Arithmetic Circuit Structure

Each wire carries a signal value, and each gate enforces arithmetic constraints

4
R1CS (Rank-1 Constraint System)

Converting to Mathematical Constraints

Circom converts circuit logic into R1CS format: (A • w) ○ (B • w) = C • w

Original Constraint
square === x * x

High-level Circom syntax

R1CS Representation
A = [0, 1, 0] // x coefficient
B = [0, 1, 0] // x coefficient
C = [1, 0, 0] // square coefficient

Matrix form for proving systems

Witness Vector
w = [square, x, 1]

All signal values including constants

Verification
(0×square + 1×x + 0×1) × (0×square + 1×x + 0×1) = 1×square + 0×x + 0×1

Mathematical verification of constraint

Complete Workflow Example

🔍 Proving Knowledge of Square Root of 25

Let's trace through a complete example:

1. Circuit Compilation
circom circuit.circom --r1cs --wasm

Generates constraint system and witness generator

2. Trusted Setup
snarkjs groth16 setup

Creates proving and verification keys

3. Input Assignment
{"x": 5, "square": 25}

Private: x=5, Public: square=25

4. Witness Generation
node generate_witness.js

Computes all intermediate signal values

5. Proof Generation
snarkjs groth16 prove

Creates zk-SNARK proof (π_a, π_b, π_c)

6. Proof Verification
snarkjs groth16 verify

Verifies proof without knowing x=5

🎉 Result:

Prover proves: "I know a number that when squared equals 25"

Verifier confirms: "Yes, the prover knows such a number"

Privacy maintained: Verifier never learns that the number is 5

🚀 The Circom Revolution

Democratizing Zero-Knowledge: Before Circom, writing zk-proofs required deep cryptographic knowledge and working with low-level mathematical constraints. Circom makes it accessible through familiar programming constructs.

The Magic Translation: Circom automatically converts high-level logic (if/else, loops, templates) into the arithmetic circuits needed for zk-SNARKs. It handles the complex math so developers can focus on application logic.
