Elliptic Curve Diffie-Hellman

Elliptic Curve Cryptography
Security: Elliptic Curve Discrete Logarithm

Algorithm Overview

Elliptic Curve Diffie-Hellman (ECDH) is a key agreement protocol that allows two parties to establish a shared secret over an insecure channel. It's based on the algebraic structure of elliptic curves over finite fields and provides the same security as traditional Diffie-Hellman but with smaller key sizes.

ECDH is widely used in modern cryptographic protocols due to its efficiency and strong security. It's particularly valuable in resource-constrained environments like mobile devices and IoT systems.

Key Generation

Each party generates a private key and computes their public key by multiplying the base point

Key Exchange

Parties exchange public keys over an insecure channel

Shared Secret

Each party computes the shared secret by combining their private key with the other party's public key

Efficiency

Provides equivalent security to RSA with significantly smaller key sizes (256-bit ECC ≈ 3072-bit RSA)

Algorithm Steps

ECDH key exchange involves the following steps to establish a shared secret:

1. Curve Selection

Both parties agree on an elliptic curve and a base point (G) on that curve. In this example: y² = x³ + 2x + 2 mod 17 with base point G=(5,1).

2. Alice's Key Generation

Alice selects a private key (a=3) and computes her public key (a×G) = 3×(5,1) = (10,6).

3. Bob's Key Generation

Bob selects a private key (b=4) and computes his public key (b×G) = 4×(5,1) = (3,1).

4. Key Exchange

Alice and Bob exchange their public keys over an insecure channel. Alice sends (10,6) to Bob, Bob sends (3,1) to Alice.

5. Alice Computes Secret

Alice computes the shared secret: a×(b×G) = 3×(3,1) = (9,16).

6. Bob Computes Secret

Bob computes the shared secret: b×(a×G) = 4×(10,6) = (9,16). Both parties now have the same shared secret.

Pseudocode

// Both parties agree on curve parameters
curve = EllipticCurve(a: 2, b: 2, p: 17)
G = Point(5, 1)  // Base point

// Alice's key generation
a = random_int(1, curve.order - 1)  // Private key
A = curve.scalar_multiply(a, G)     // Public key

// Bob's key generation
b = random_int(1, curve.order - 1)  // Private key
B = curve.scalar_multiply(b, G)     // Public key

// Exchange public keys
alice_receives(B)
bob_receives(A)

// Shared secret computation
// Alice:
shared_secret_alice = curve.scalar_multiply(a, B)

// Bob:
shared_secret_bob = curve.scalar_multiply(b, A)

// Both secrets should be equal: (a×b)×G

Applications

ECDH is widely used in modern security protocols and systems:

  • TLS/SSL for secure web browsing (ECDHE cipher suites)
  • Secure messaging apps (Signal, WhatsApp)
  • VPN protocols (WireGuard, IKEv2)
  • Blockchain and cryptocurrency security
  • Secure shell (SSH) key exchange
  • Mobile device security (iOS, Android)
