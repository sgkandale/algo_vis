ElGamal Key Exchange

Public-Key Cryptography
Security: Discrete Logarithm Problem

Algorithm Overview

ElGamal encryption is a public-key cryptosystem based on the discrete logarithm problem. It consists of three components: key generation, encryption, and decryption. The system provides an alternative to RSA and is widely used in cryptographic protocols.

In key exchange scenarios, ElGamal allows two parties to establish a shared secret over an insecure channel. The algorithm is particularly valued for its security properties and its basis in the difficulty of computing discrete logarithms in a finite field.

Key Generation

Select large prime, generator, and private/public keys

Encryption

Encrypt using public key and random ephemeral key

Decryption

Decrypt using private key and shared parameters

Security

Based on difficulty of discrete logarithm problem

Algorithm Steps

ElGamal key exchange involves the following steps to securely share a symmetric key:

1. Key Generation

Alice selects a large prime (p=23), a generator (g=11), and a private key (x=6). She computes her public key h = g^x mod p = 11^6 mod 23 = 9.

2. Share Public Key

Alice shares her public key (p, g, h) with Bob. This key can be safely transmitted over insecure channels.

3. Generate Ephemeral Key

Bob generates a random ephemeral key (y=3) that will be used only for this key exchange session.

4. Encrypt Symmetric Key

Bob encrypts the symmetric key (K=12) using Alice's public key: c1 = g^y mod p = 11^3 mod 23 = 20, s = h^y mod p = 9^3 mod 23 = 16, c2 = K·s mod p = 12·16 mod 23 = 8.

5. Send Encrypted Pair

Bob sends the encrypted pair (c1, c2) = (20, 8) to Alice. This value cannot be decrypted without the private key.

6. Decrypt Key

Alice decrypts the symmetric key using her private key: s = c1^x mod p = 20^6 mod 23 = 16, K = c2·s^{-1} mod p = 8·13 mod 23 = 12.

Pseudocode

// Key Generation
p = large_prime()
g = generator_mod_p()
x = random_int(1, p-2)  // Private key
h = g^x mod p           // Public key

// Public Key: (p, g, h)
// Private Key: x

// Encryption (by Bob)
y = random_int(1, p-2)  // Ephemeral key
c1 = g^y mod p
s = h^y mod p
c2 = (K * s) mod p      // K is symmetric key

// Send (c1, c2) to Alice

// Decryption (by Alice)
s = c1^x mod p
s_inv = modular_inverse(s, p)
K = (c2 * s_inv) mod p
