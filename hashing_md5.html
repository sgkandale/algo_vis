MD5 (Message Digest 5) is a widely used hash function producing a 128-bit hash value. Despite being cryptographically broken, it's still used for checksums and data integrity verification in non-security contexts.

The algorithm processes input in 512-bit blocks through 64 rounds of operations using four auxiliary functions, producing a fixed-size output regardless of input length.

Fast Computation

Designed for speed, MD5 can process large amounts of data quickly, making it suitable for checksums

Fixed Output

Always produces a 128-bit (16-byte) hash regardless of input size, from single characters to large files

Four Functions

Uses four different auxiliary functions (F, G, H, I) across 64 rounds of processing

Broken Security

Collision attacks have been demonstrated, making it unsuitable for cryptographic security applications

Current Applications

While MD5 is no longer secure for cryptographic purposes, it still has legitimate uses:

  • File integrity checksums (non-security)
  • Database record deduplication
  • Cache keys and identifiers
  • Legacy system compatibility
  • Educational and research purposes
