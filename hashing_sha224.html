SHA-224

Cryptographic Hash Algorithm
Output: 224 bits (56 characters)

Algorithm Overview

SHA-224 (Secure Hash Algorithm 224) is a cryptographic hash function that is part of the SHA-2 family. It produces a 224-bit (28-byte) hash value, typically rendered as a 56-digit hexadecimal number. SHA-224 is identical to SHA-256, except with different initial hash values and truncated output.

SHA-224 is commonly used in security protocols and applications, including TLS and SSL, PGP, SSH, and IPsec. It provides a high level of security and is resistant to known cryptographic attacks, though it's being gradually replaced by SHA-3 in newer applications.

Key Feature

Produces a fixed-size 224-bit output from variable-size input

Performance

Efficient implementation with good performance on most platforms

Security

Resistant to collision attacks, preimage attacks, and second preimage attacks

Structure

Merkle-Damgård construction with Davies-Meyer compression function

Algorithm Steps

SHA-224 processes input data through several stages to produce a fixed-size hash:

1. Pre-processing

Pad the message to ensure its length is congruent to 448 modulo 512. Append the original message length as a 64-bit big-endian integer.

2. Parsing

Break the padded message into 512-bit blocks that will be processed sequentially.

3. Hash Computation

Initialize hash values with specific constants. Process each 512-bit block through 64 rounds of compression functions.

4. Compression

Each round uses logical functions and constant values to mix the bits thoroughly.

5. Finalization

After processing all blocks, the final hash value is produced by truncating the 256-bit output to 224 bits.

Pseudocode

// SHA-224 Pseudocode
function sha224(message):
    // Initialize variables with SHA-224 specific initial hash values
    h0 = 0xc1059ed8
    h1 = 0x367cd507
    h2 = 0x3070dd17
    h3 = 0xf70e5939
    h4 = 0xffc00b31
    h5 = 0x68581511
    h6 = 0x64f98fa7
    h7 = 0xbefa4fa4

    // Pre-processing: padding the message
    padded_message = pad_message(message)

    // Process the message in successive 512-bit chunks
    for each 512-bit chunk of padded_message:
        // Prepare message schedule array
        w[0..63] = create_message_schedule(chunk)

        // Initialize working variables with current hash values
        a = h0
        b = h1
        c = h2
        d = h3
        e = h4
        f = h5
        g = h6
        h = h7

        // Main compression function
        for i from 0 to 63:
            S1 = (e rightrotate 6) XOR (e rightrotate 11) XOR (e rightrotate 25)
            ch = (e AND f) XOR ((NOT e) AND g)
            temp1 = h + S1 + ch + k[i] + w[i]
            S0 = (a rightrotate 2) XOR (a rightrotate 13) XOR (a rightrotate 22)
            maj = (a AND b) XOR (a AND c) XOR (b AND c)
            temp2 = S0 + maj

            h = g
            g = f
            f = e
            e = d + temp1
            d = c
            c = b
            b = a
            a = temp1 + temp2

        // Add the compressed chunk to the current hash value
        h0 = h0 + a
        h1 = h1 + b
        h2 = h2 + c
        h3 = h3 + d
        h4 = h4 + e
        h5 = h5 + f
        h6 = h6 + g
        h7 = h7 + h

    // Produce the final hash value (224 bits)
    digest = h0 concatenated with h1 concatenated with h2 concatenated with h3
             concatenated with h4 concatenated with h5 concatenated with h6

    return digest.toHexString()

function pad_message(message):
    // Append the bit '1' to the message
    // Append k '0' bits, where k is the smallest non-negative solution to
    // (message length + 1 + k) ≡ 448 mod 512
    // Append length of original message (without padding), as a 64-bit big-endian integer

function create_message_schedule(chunk):
    // Break chunk into sixteen 32-bit big-endian words w[0..15]
    // Extend the sixteen 32-bit words into sixty-four 32-bit words
    for i from 16 to 63:
        s0 = (w[i-15] rightrotate 7) XOR (w[i-15] rightrotate 18) XOR (w[i-15] rightshift 3)
        s1 = (w[i-2] rightrotate 17) XOR (w[i-2] rightrotate 19) XOR (w[i-2] rightshift 10)
        w[i] = w[i-16] + s0 + w[i-7] + s1

Implementation

Below is a JavaScript implementation of the SHA-224 hash algorithm. This implementation follows the official specification and produces correct SHA-224 hashes.

JavaScript Implementation 
function sha224(message) {
  // Convert message to UTF-8 bytes
  const msgBuffer = new TextEncoder().encode(message);

  // SHA-224 initial hash values
  let h0 = 0xc1059ed8, h1 = 0x367cd507, h2 = 0x3070dd17;
  let h3 = 0xf70e5939, h4 = 0xffc00b31, h5 = 0x68581511;
  let h6 = 0x64f98fa7, h7 = 0xbefa4fa4;

  // Pre-processing: padding
  const bitLen = msgBuffer.length * 8;
  const padLen = (((bitLen + 64) >> 9) << 6) + 448 - (bitLen % 512);
  const totalLen = bitLen + padLen + 64;

  // Create a buffer for the padded message
  const totalBytes = totalLen >> 3;
  const padded = new Uint8Array(totalBytes);
  padded.set(msgBuffer, 0);
  padded[msgBuffer.length] = 0x80; // Append 1 bit (and the rest zeros)

  // Append length as 64-bit big-endian integer
  const lenPos = totalBytes - 8;
  const bitLenHi = (bitLen / 0x100000000) >>> 0;
  const bitLenLo = bitLen >>> 0;
  padded[lenPos] = (bitLenHi >> 24) & 0xff;
  padded[lenPos + 1] = (bitLenHi >> 16) & 0xff;
  padded[lenPos + 2] = (bitLenHi >> 8) & 0xff;
  padded[lenPos + 3] = bitLenHi & 0xff;
  padded[lenPos + 4] = (bitLenLo >> 24) & 0xff;
  padded[lenPos + 5] = (bitLenLo >> 16) & 0xff;
  padded[lenPos + 6] = (bitLenLo >> 8) & 0xff;
  padded[lenPos + 7] = bitLenLo & 0xff;

  // Process each 512-bit chunk
  for (let i = 0; i < totalBytes; i += 64) {
    // Create message schedule array
    const w = new Array(64);

    // Copy chunk into first 16 words
    for (let j = 0; j < 16; j++) {
      const offset = i + j * 4;
      w[j] = (padded[offset] << 24) | (padded[offset + 1] << 16) |
             (padded[offset + 2] << 8) | padded[offset + 3];
    }

    // Extend the first 16 words into the remaining 48 words
    for (let j = 16; j < 64; j++) {
      const s0 = ((w[j-15] >>> 7) | (w[j-15] << (32 - 7))) ^
                 ((w[j-15] >>> 18) | (w[j-15] << (32 - 18))) ^
                 (w[j-15] >>> 3);
      const s1 = ((w[j-2] >>> 17) | (w[j-2] << (32 - 17))) ^
                 ((w[j-2] >>> 19) | (w[j-2] << (32 - 19))) ^
                 (w[j-2] >>> 10);
      w[j] = (w[j-16] + s0 + w[j-7] + s1) >>> 0;
    }

    // Initialize working variables
    let a = h0, b = h1, c = h2, d = h3;
    let e = h4, f = h5, g = h6, hVal = h7;

    // Compression function main loop
    for (let j = 0; j < 64; j++) {
      const S1 = ((e >>> 6) | (e << (32 - 6))) ^
                 ((e >>> 11) | (e << (32 - 11))) ^
                 ((e >>> 25) | (e << (32 - 25)));
      const ch = (e & f) ^ ((~e) & g);
      const temp1 = (hVal + S1 + ch + K[j] + w[j]) >>> 0;
      const S0 = ((a >>> 2) | (a << (32 - 2))) ^
                 ((a >>> 13) | (a << (32 - 13))) ^
                 ((a >>> 22) | (a << (32 - 22)));
      const maj = (a & b) ^ (a & c) ^ (b & c);
      const temp2 = (S0 + maj) >>> 0;

      hVal = g;
      g = f;
      f = e;
      e = (d + temp1) >>> 0;
      d = c;
      c = b;
      b = a;
      a = (temp1 + temp2) >>> 0;
    }

    // Add the compressed chunk to the current hash value
    h0 = (h0 + a) >>> 0;
    h1 = (h1 + b) >>> 0;
    h2 = (h2 + c) >>> 0;
    h3 = (h3 + d) >>> 0;
    h4 = (h4 + e) >>> 0;
    h5 = (h5 + f) >>> 0;
    h6 = (h6 + g) >>> 0;
    h7 = (h7 + hVal) >>> 0;
  }

  // Produce the final hash value (224 bits)
  const hashArray = [
    h0 >> 24 & 0xff, h0 >> 16 & 0xff, h0 >> 8 & 0xff, h0 & 0xff,
    h1 >> 24 & 0xff, h1 >> 16 & 0xff, h1 >> 8 & 0xff, h1 & 0xff,
    h2 >> 24 & 0xff, h2 >> 16 & 0xff, h2 >> 8 & 0xff, h2 & 0xff,
    h3 >> 24 & 0xff, h3 >> 16 & 0xff, h3 >> 8 & 0xff, h3 & 0xff,
    h4 >> 24 & 0xff, h4 >> 16 & 0xff, h4 >> 8 & 0xff, h4 & 0xff,
    h5 >> 24 & 0xff, h5 >> 16 & 0xff, h5 >> 8 & 0xff, h5 & 0xff,
    h6 >> 24 & 0xff, h6 >> 16 & 0xff, h6 >> 8 & 0xff, h6 & 0xff
  ];

  // Convert to hex string
  return hashArray.map(b => b.toString(16).padStart(2, '0')).join('');
}

Applications

SHA-224 is used in various security applications and protocols including:

  • Digital signatures and certificate authorities
  • TLS and SSL security protocols
  • SSH (Secure Shell) protocol
  • IPsec and VPN technologies
  • Blockchain and cryptocurrency technologies
  • Password hashing (with salt)

Pros & Cons

Advantages

High security level, standardized algorithm, fast computation, resistance to known cryptographic attacks, widely supported

Disadvantages

Vulnerable to length-extension attacks, being gradually replaced by SHA-3, not as fast as some non-cryptographic hashes for non-security purposes

SHA Family

SHA-224 is part of the SHA-2 family of cryptographic hash functions:

  • SHA-224: 224-bit output, truncated version of SHA-256
  • SHA-256: 256-bit output, most widely used SHA-2 variant
  • SHA-384: 384-bit output, truncated version of SHA-512
  • SHA-512: 512-bit output, designed for 64-bit platforms
  • SHA-512/224, SHA-512/256: Truncated versions of SHA-512
