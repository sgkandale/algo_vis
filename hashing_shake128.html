SHAKE128

Extendable-Output Function (XOF)
Security Strength: 128 bits

Algorithm Overview

SHAKE128 is an extendable-output function (XOF) from the SHA-3 family. Unlike fixed-length hash functions, SHAKE128 can produce output of any desired length, making it extremely versatile for various cryptographic applications.

SHAKE128 is based on the same Keccak sponge construction as other SHA-3 algorithms but is designed to support arbitrary output lengths. It provides 128 bits of security against collision attacks and is particularly useful for applications that require variable-length output like key derivation, stream encryption, and randomized algorithms.

Key Feature

Extendable output - can generate output of any desired length

Performance

Efficient in both hardware and software implementations

Security

Provides 128-bit security against collision and preimage attacks

Structure

Uses Keccak sponge construction with a state of 1600 bits

Algorithm Steps

SHAKE128 processes input data through several stages using the sponge construction:

1. Message Padding

The message is padded with the pattern 1111 followed by the SHAKE-specific padding rule.

2. Absorption Phase

The padded message is divided into blocks and XORed into the state, followed by applying the Keccak-f permutation.

3. Keccak-f Permutation

The 1600-bit state undergoes 24 rounds of permutation, each consisting of five steps: θ, ρ, π, χ, and ι.

4. Squeezing Phase

Output bits are extracted from the state. Unlike fixed-length hashes, squeezing continues until the desired output length is achieved.

5. Extendable Output

The output can be extended to any desired length by continuing the squeezing process.

        // XOR the block into the state
        for i from 0 to (rate/64 - 1):
            x = i % 5
            y = floor(i / 5)
            state[x][y] = state[x][y] XOR block[i]

        // Apply Keccak-f[1600] permutation
        state = keccak_f_1600(state)

    // Squeezing phase - continues until desired output length
    output = ""
    while output.length < outputLength:
        // Extract output from the state
        for i from 0 to (rate/64 - 1):
            x = i % 5
            y = floor(i / 5)
            output = output + state[x][y]

        if output.length < outputLength:
            state = keccak_f_1600(state)

    // Return the requested output length
    return output[0:outputLength]

Applications

SHAKE128 is used in various security applications and protocols including:

  • Key derivation functions
  • Deterministic random bit generation
  • Stream encryption
  • Password-based cryptography
  • Protocol randomization
  • Digital signatures (as part of other constructions)

Pros & Cons

Advantages

Variable output length, high security, resistance to length-extension attacks, efficient implementation, standardized algorithm

Disadvantages

Relatively new compared to older algorithms, may not be supported in all cryptographic libraries, slower than some non-cryptographic RNGs for simple applications

SHA-3 XOF Family

SHAKE128 is part of the SHA-3 family of extendable-output functions:

  • SHAKE128: 128-bit security, rate=1344, capacity=256
  • SHAKE256: 256-bit security, rate=1088, capacity=512
  • SHA3-224/256/384/512: Fixed-length hash functions
  • cSHAKE: Customizable variant of SHAKE
