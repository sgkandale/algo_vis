SHAKE256

Cryptographic Extendable-Output Function (XOF)
Security Strength: 256-bit

Algorithm Overview

SHAKE256 is a member of the SHA-3 (Secure Hash Algorithm 3) family of cryptographic functions. It is an extendable-output function (XOF) that can produce output of any desired length, based on the SHA-3 framework.

Unlike traditional hash functions with fixed output sizes, SHAKE256 uses the sponge construction to absorb input data and then squeeze output bits in a continuous stream. This makes it particularly useful for applications requiring variable-length output like key derivation, stream encryption, and generating random bits.

Key Feature

Extendable output function with arbitrary output length

Performance

Based on Keccak sponge function with 24 rounds of permutation

Internal State

1600-bit internal state with 1088-bit absorption rate (r) and 512-bit capacity (c)

Construction

Sponge construction with Keccak-f[1600] permutation function

Algorithm Steps

SHAKE256 follows the sponge construction paradigm with these main steps:

1. Padding

The input message is padded with the suffix "1111" followed by a "1", then zeros, and finally a "1" to make the length a multiple of the rate (1088 bits).

2. Absorption Phase

The padded message is divided into blocks of r bits (1088 bits). Each block is XORed into the state, followed by application of the Keccak-f permutation function.

3. Squeezing Phase

After processing all input blocks, output bits are extracted from the state in blocks of r bits. If more output is needed, the permutation function is applied again.

4. Output Truncation

The output can be truncated to the desired length. For SHAKE256, the security strength is 256 bits, meaning it provides 256-bit security against collisions and preimage attacks.

Pseudocode

function SHAKE256(input, outputLength):
    # Initialize state
    state = 0^{1600}

    # Padding
    padded = input || 1111 || 0* || 1  # Until length is multiple of 1088

    # Absorption phase
    for each block in padded:
        state = state XOR block
        state = Keccak-f[1600](state)

    # Squeezing phase
    output = ""
    while output length < outputLength:
        output = output || first 1088 bits of state
        if output length < outputLength:
            state = Keccak-f[1600](state)

    return first outputLength bits of output

Implementation

Below is a Python implementation of SHAKE256 using the standard library. In practice, you should use established cryptographic libraries rather than implementing your own for security-critical applications.

Python Implementation 
import hashlib

# Using Python's built-in library implementation
def shake256(input_data, output_length):
    # Create a SHAKE256 hash object
    shake = hashlib.shake_256()

    # Update with input data (bytes)
    if isinstance(input_data, str):
        input_data = input_data.encode('utf-8')
    shake.update(input_data)

    # Generate output of desired length
    return shake.hexdigest(output_length)

# Example usage
input_text = "Hello, World!"
output_length = 64  # 64 hex characters = 32 bytes

hash_result = shake256(input_text, output_length)
print(f"SHAKE256 hash of '{input_text}':")
print(hash_result)

Applications

SHAKE256 is used in various cryptographic applications where variable-length output is needed:

  • Key derivation functions (KDFs)
  • Deterministic random bit generators (DRBGs)
  • Message authentication codes
  • Digital signatures and certificates
  • Cryptographic protocols requiring extensible output
