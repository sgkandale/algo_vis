How FIFO Works

Concept

The First-In-First-Out (FIFO) page replacement algorithm replaces the page that has been in memory the longest.

It treats page frames as a circular buffer and replaces pages in the order they were loaded.

Implementation

FIFO can be implemented using:

  • A queue to track page insertion order
  • A pointer that cycles through frames
  • A timestamp for each page

When a page needs to be replaced, the oldest page is removed.

Advantages

  • Simple to understand and implement
  • Low overhead with queue implementation
  • Fairness - all pages get equal residency time
  • No per-page tracking required

Disadvantages

  • Poor performance in many cases
  • Affected by Belady's Anomaly (more frames can lead to more faults)
  • Doesn't consider page usage frequency
  • May remove frequently used pages
