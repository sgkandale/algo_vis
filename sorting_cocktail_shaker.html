Cocktail Shaker Sort

Sorting Algorithm
Time Complexity: O(n) to O(n²)

Algorithm Overview

Cocktail Shaker Sort, also known as bidirectional bubble sort, cocktail sort, shaker sort, ripple sort, or shuttle sort, is a variation of the bubble sort algorithm. It differs by sorting in both directions on each pass through the list.

The algorithm gets its name from the way it moves elements - similar to the motion of a cocktail shaker. It addresses the issue of bubble sort's "turtle" problem, where small elements at the end of the list take many passes to move to the beginning.

Key Feature

Sorts in both directions, improving on bubble sort

Performance

O(n) in best case (already sorted), O(n²) average and worst case

Space

O(1) additional space (in-place algorithm)

Approach

Bidirectional bubbling of elements

Algorithm Steps

Cocktail Shaker Sort alternates between two directions on each pass:

1. Forward Pass

Traverse from left to right, comparing adjacent elements and swapping if they are in the wrong order

2. Backward Pass

Traverse from right to left, comparing adjacent elements and swapping if they are in the wrong order

3. Adjust Boundaries

After each pass, adjust the boundaries since the largest/smallest element has bubbled to the end

4. Repeat

Continue alternating directions until no swaps occur in a full pass

5. Completion

When a complete pass with no swaps occurs, the list is sorted

Pseudocode

procedure cocktailShakerSort(A: list of sortable items)
    n = length(A)
    swapped = true
    start = 0
    end = n - 1
    
    while swapped is true
        // Reset the swapped flag
        swapped = false
        
        // Forward pass (left to right)
        for i = start to end - 1
            if A[i] > A[i + 1] then
                swap(A[i], A[i + 1])
                swapped = true
        
        // If nothing moved, array is sorted
        if not swapped then break
        
        // Otherwise, reset the swapped flag
        swapped = false
        
        // Move the end point back by one
        end = end - 1
        
        // Backward pass (right to left)
        for i = end - 1 downto start
            if A[i] > A[i + 1] then
                swap(A[i], A[i + 1])
                swapped = true
        
        // Move the start point forward by one
        start = start + 1

Implementation

Below is a Python implementation of Cocktail Shaker Sort. The algorithm alternates between forward and backward passes until the list is sorted.

Python Implementation 
def cocktail_shaker_sort(arr):
    if len(arr) <= 1:
        return arr
        
    swapped = True
    start = 0
    end = len(arr) - 1
    
    while swapped:
        # Reset swapped flag at start of each pass
        swapped = False
        
        # Forward pass (left to right)
        for i in range(start, end):
            if arr[i] > arr[i + 1]:
                # Swap elements
                arr[i], arr[i + 1] = arr[i + 1], arr[i]
                swapped = True
        
        # If no swaps, array is sorted
        if not swapped:
            break
        
        # Otherwise, reset swapped flag
        swapped = False
        
        # Move end point backward
        end -= 1
        
        # Backward pass (right to left)
        for i in range(end - 1, start - 1, -1):
            if arr[i] > arr[i + 1]:
                # Swap elements
                arr[i], arr[i + 1] = arr[i + 1], arr[i]
                swapped = True
        
        # Move start point forward
        start += 1
    
    return arr

# Example usage
arr = [38, 27, 43, 3, 9, 82, 10, 15, 22]
sorted_arr = cocktail_shaker_sort(arr)
print("Sorted array:", sorted_arr)
# Output: Sorted array: [3, 9, 10, 15, 22, 27, 38, 43, 82]

Applications

Cocktail Shaker Sort is useful in specific scenarios:

  • Educational purposes to demonstrate bidirectional sorting
  • Situations where data is nearly sorted
  • Small datasets
  • When a stable sort is required
  • As an improvement to bubble sort

Advantages

Compared to regular bubble sort:

  • More efficient for certain data distributions
  • Solves the "turtle problem" of bubble sort
  • Better performance on nearly sorted lists
  • Maintains bubble sort's simplicity
