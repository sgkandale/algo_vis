Counting Sort

Non-Comparison Sorting Algorithm
Time Complexity: O(n + k)

Algorithm Overview

Counting Sort is an efficient, non-comparison based sorting algorithm that works by counting the occurrences of each distinct element in the input array. It is particularly useful when the range of potential items (k) is not significantly larger than the number of items (n).

Unlike comparison-based algorithms (like QuickSort or MergeSort) that have a lower bound of O(n log n), Counting Sort operates in linear time O(n + k), making it very efficient for the right use cases. However, it requires additional memory proportional to the range of the data.

Key Feature

Linear time complexity when range is not substantially larger than n

Performance

O(n + k) time complexity (n = elements, k = range)

Space

Requires O(n + k) additional space

Approach

Count frequencies, calculate positions, place elements in output

Algorithm Steps

Counting Sort works in three main phases: counting frequencies, calculating positions, and building the output array.

1. Find Maximum

Determine the maximum value in the input array to know the range of values.

2. Count Frequencies

Create a counting array and count the occurrences of each value in the input.

3. Calculate Positions

Modify the counting array to store cumulative counts (starting positions).

4. Build Output

Place each element from the input array into its correct position in the output array.

5. Copy to Original

Copy the sorted output back to the original array (optional step).

Pseudocode

function countingSort(arr):
    n = length(arr)
    max_val = findMax(arr)  // Find maximum value in array
    
    // Create counting array of size max_val+1
    count = new Array(max_val + 1) filled with 0
    
    // Count frequencies of each element
    for i from 0 to n-1:
        count[arr[i]] += 1
    
    // Modify count array to store cumulative counts
    for i from 1 to max_val:
        count[i] += count[i - 1]
    
    // Build the output array
    output = new Array(n)
    for i from n-1 downto 0:
        output[count[arr[i]] - 1] = arr[i]
        count[arr[i]] -= 1
    
    // Copy output to original array
    for i from 0 to n-1:
        arr[i] = output[i]

Implementation

Below is a Python implementation of Counting Sort. The algorithm efficiently sorts integers by counting occurrences and placing elements directly in their sorted positions.

Python Implementation 
def counting_sort(arr):
    # Find the maximum element
    max_val = max(arr)
    
    # Initialize count array with zeros
    count = [0] * (max_val + 1)
    
    # Count frequencies
    for num in arr:
        count[num] += 1
    
    # Calculate cumulative counts
    for i in range(1, len(count)):
        count[i] += count[i - 1]
    
    # Build the output array
    output = [0] * len(arr)
    # Traverse from end to maintain stability
    for i in range(len(arr) - 1, -1, -1):
        num = arr[i]
        pos = count[num] - 1
        output[pos] = num
        count[num] -= 1
    
    # Copy output to original array
    for i in range(len(arr)):
        arr[i] = output[i]

# Example usage
arr = [4, 2, 2, 8, 3, 3, 1]
counting_sort(arr)
print("Sorted array:", arr)  # Output: [1, 2, 2, 3, 3, 4, 8]

Applications

Counting Sort is particularly effective in specific scenarios with limited value ranges:

  • Sorting small integer ranges (e.g., exam scores 0-100)
  • As a subroutine in Radix Sort for larger integers
  • Histogram generation and frequency analysis
  • Applications requiring stable sorting of integers
  • Data with limited discrete values (e.g., grades A-F)

Advantages & Disadvantages

Advantages

  • Linear time complexity O(n + k)
  • Stable sorting algorithm
  • Efficient for small integer ranges
  • Easy to implement and understand

Disadvantages

  • Only works with integer keys
  • Inefficient for large value ranges (k)
  • Requires additional memory O(n + k)
  • Not suitable for floating point numbers

Complexity Analysis

Counting Sort has a consistent time complexity across all cases:

Time Complexity: O(n + k)
Space Complexity: O(n + k)

Where n is the number of elements and k is the range of input values. When k is O(n), Counting Sort runs in linear time O(n).
