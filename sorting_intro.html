IntroSort

Hybrid Sorting Algorithm
Time Complexity: O(n log n)

Algorithm Overview

IntroSort (Introspective Sort) is a hybrid sorting algorithm that combines the best features of Quicksort, Heapsort, and Insertion Sort. It begins with Quicksort and switches to Heapsort when the recursion depth exceeds a level based on the number of elements. For small subarrays, it switches to Insertion Sort for better performance.

Developed by David Musser in 1997, IntroSort provides both fast average performance of Quicksort and optimal worst-case performance of O(n log n). It's widely used in standard libraries like the C++ Standard Template Library (STL).

Key Feature

Hybrid approach with optimal worst-case performance

Performance

O(n log n) worst-case, O(n) best-case (nearly sorted)

Space

O(log n) for recursion stack (in-place)

Approach

Quicksort + Heapsort + Insertion Sort hybrid

Algorithm Steps

IntroSort intelligently combines three sorting algorithms:

1. Quicksort Phase

Starts with Quicksort using median-of-three pivot selection

2. Recursion Limit

Switches to Heapsort if recursion depth exceeds 2*log(n)

3. Heapsort Phase

Uses Heapsort when worst-case Quicksort behavior is detected

4. Insertion Sort

Switches to Insertion Sort for small subarrays (≤ 16 elements)

Pseudocode

procedure introsort(array, depthLimit)
    n = length(array)
    
    if n <= 16:
        insertionSort(array)
    else if depthLimit == 0:
        heapsort(array)
    else:
        pivot = medianOfThree(array)
        partitionIndex = partition(array, pivot)
        
        introsort(array[0:partitionIndex], depthLimit - 1)
        introsort(array[partitionIndex+1:n], depthLimit - 1)

procedure sort(array)
    depthLimit = 2 * floor(log2(length(array)))
    introsort(array, depthLimit)

Implementation

Below is a Python implementation of the IntroSort algorithm. The implementation includes all three phases: Quicksort, Heapsort, and Insertion Sort.

Python Implementation 
import math

def insertion_sort(arr, start=0, end=None):
    if end is None:
        end = len(arr) - 1
        
    for i in range(start + 1, end + 1):
        key = arr[i]
        j = i - 1
        while j >= start and arr[j] > key:
            arr[j + 1] = arr[j]
            j -= 1
        arr[j + 1] = key

def heapify(arr, n, i):
    largest = i
    left = 2 * i + 1
    right = 2 * i + 2
    
    if left < n and arr[left] > arr[largest]:
        largest = left
        
    if right < n and arr[right] > arr[largest]:
        largest = right
        
    if largest != i:
        arr[i], arr[largest] = arr[largest], arr[i]
        heapify(arr, n, largest)

def heapsort(arr, start=0, end=None):
    if end is None:
        end = len(arr) - 1
        
    n = end - start + 1
    
    # Build max heap
    for i in range(start + n // 2 - 1, start - 1, -1):
        heapify(arr, n, i)
    
    # Extract elements from heap
    for i in range(end, start, -1):
        arr[start], arr[i] = arr[i], arr[start]
        heapify(arr, i - start, start)

def partition(arr, low, high):
    # Median-of-three pivot selection
    mid = (low + high) // 2
    if arr[mid] < arr[low]:
        arr[low], arr[mid] = arr[mid], arr[low]
    if arr[high] < arr[low]:
        arr[low], arr[high] = arr[high], arr[low]
    if arr[mid] < arr[high]:
        arr[mid], arr[high] = arr[high], arr[mid]
        
    pivot = arr[high]
    i = low - 1
    
    for j in range(low, high):
        if arr[j] <= pivot:
            i += 1
            arr[i], arr[j] = arr[j], arr[i]
            
    arr[i + 1], arr[high] = arr[high], arr[i + 1]
    return i + 1

def introsort(arr, start=0, end=None, depth_limit=None):
    if end is None:
        end = len(arr) - 1
    if depth_limit is None:
        depth_limit = 2 * math.floor(math.log2(end - start + 1))
        
    size = end - start + 1
    
    if size <= 16:
        insertion_sort(arr, start, end)
        return
        
    if depth_limit == 0:
        heapsort(arr, start, end)
        return
        
    pivot_index = partition(arr, start, end)
    introsort(arr, start, pivot_index - 1, depth_limit - 1)
    introsort(arr, pivot_index + 1, end, depth_limit - 1)

def intro_sort(arr):
    introsort(arr)
    return arr

# Example usage
arr = [8, 3, 2, 7, 4, 9, 1, 6, 5]
sorted_arr = intro_sort(arr)
print("Sorted array:", sorted_arr)

Applications

IntroSort is widely used in production systems due to its efficiency and reliability:

  • C++ Standard Template Library (STL) sort() function
  • Java's Arrays.sort() for primitive types
  • Python's sorted() and list.sort() functions
  • Database management systems for sorting operations
  • High-performance computing applications

Complexity Analysis

IntroSort has a worst-case time complexity of O(n log n), which is optimal for comparison-based sorting. This is achieved by switching to Heapsort when Quicksort's recursion depth becomes too deep.

The average-case performance is O(n log n), similar to Quicksort. For small arrays (≤16 elements), it uses Insertion Sort which has O(n²) complexity but is faster for small n due to low constant factors.

Advantages

Optimal worst-case performance: Guarantees O(n log n) time complexity in all cases.

Efficient for large datasets: Combines Quicksort's speed with Heapsort's reliability.

Adaptive: Uses Insertion Sort for small subarrays where it's most efficient.
