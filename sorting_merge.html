Merge Sort

Sorting Algorithm
Time Complexity: O(n log n)

Algorithm Overview

Merge Sort is an efficient, stable, comparison-based, divide and conquer sorting algorithm. It works by recursively dividing the unsorted list into n sublists until each sublist contains one element (a list of one element is considered sorted), then repeatedly merging sublists to produce new sorted sublists until there is only one sublist remaining.

Merge Sort is particularly useful when sorting large datasets and is often used for external sorting. It's a classic example of a divide and conquer algorithm and is widely used in practice due to its efficiency and stability.

Key Feature

Stable sorting algorithm that preserves the order of equal elements

Performance

Consistent O(n log n) time complexity in all cases (best, average, worst)

Space

Requires O(n) additional space for temporary arrays during merging

Approach

Divide and conquer technique with recursive implementation

Algorithm Steps

Merge Sort involves the following steps to sort an array:

1. Divide

Recursively divide the unsorted array into two halves until each subarray has only one element

2. Conquer

Sort each subarray (a single element is already sorted)

3. Merge

Merge the sorted subarrays back together to produce new sorted subarrays

4. Recursive Merge

Continue merging subarrays until the entire array is sorted

Pseudocode

function mergeSort(array)
    if length(array) ≤ 1
        return array
    
    // Divide the array into two halves
    mid = length(array) / 2
    left = array[0..mid-1]
    right = array[mid..end]
    
    // Recursively sort each half
    left = mergeSort(left)
    right = mergeSort(right)
    
    // Merge the sorted halves
    return merge(left, right)

function merge(left, right)
    result = []
    i = 0, j = 0
    
    // Traverse both arrays and insert smaller value
    while i < length(left) and j < length(right)
        if left[i] ≤ right[j]
            append left[i] to result
            i = i + 1
        else
            append right[j] to result
            j = j + 1
    
    // Append remaining elements
    append remaining elements of left to result
    append remaining elements of right to result
    
    return result

Implementation

Below is a Python implementation of the Merge Sort algorithm. The algorithm consists of two main functions: merge_sort() which handles the recursive division, and merge() which combines two sorted halves into a single sorted array.

Python Implementation 
def merge_sort(arr):
    if len(arr) <= 1:
        return arr
        
    # Find the middle of the array
    mid = len(arr) // 2
    
    # Divide the array into two halves
    left_half = arr[:mid]
    right_half = arr[mid:]
    
    # Recursively sort each half
    left_half = merge_sort(left_half)
    right_half = merge_sort(right_half)
    
    # Merge the sorted halves
    return merge(left_half, right_half)

def merge(left, right):
    merged = []
    left_index = right_index = 0
    
    # Traverse both lists and insert smaller value
    while left_index < len(left) and right_index < len(right):
        if left[left_index] < right[right_index]:
            merged.append(left[left_index])
            left_index += 1
        else:
            merged.append(right[right_index])
            right_index += 1
    
    # Append remaining elements from left or right
    while left_index < len(left):
        merged.append(left[left_index])
        left_index += 1
        
    while right_index < len(right):
        merged.append(right[right_index])
        right_index += 1
        
    return merged

# Example usage
arr = [38, 27, 43, 3, 9, 82, 10]
sorted_arr = merge_sort(arr)
print("Sorted array:", sorted_arr)

Applications

Merge Sort is used in various scenarios where stable, efficient sorting is required:

  • Sorting linked lists (efficient implementation)
  • External sorting for large datasets
  • Inversion count problem
  • Used as a building block in more complex algorithms
  • Database systems for sorting large datasets

Complexity Analysis

Merge Sort has consistent time complexity in all cases:

Divide Phase

Dividing the array takes O(log n) time

Merge Phase

Merging n elements at each level takes O(n) time

Total Time

O(n log n) for all cases (best, average, worst)
