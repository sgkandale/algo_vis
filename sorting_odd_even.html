Odd-Even Sort

Sorting Algorithm
Time Complexity: O(n) to O(n²)

Algorithm Overview

Odd-Even Sort (also known as Brick Sort) is a comparison-based sorting algorithm that works by repeatedly comparing and swapping adjacent elements in two alternating phases: the odd phase and the even phase.

The algorithm is similar to bubble sort but with improved parallelization capabilities. It alternates between two passes: one for odd indices and one for even indices, allowing multiple comparisons to occur simultaneously in parallel implementations.

Key Feature

Alternates between odd and even comparison phases

Performance

O(n) in best case (already sorted), O(n²) average and worst case

Space

O(1) additional space (in-place algorithm)

Approach

Compares and swaps adjacent elements in two alternating phases

Algorithm Steps

Odd-Even Sort works in two alternating phases until the array is sorted:

1. Odd Phase

Compare and swap elements at odd indices with their next neighbors (i = 1, 3, 5,...)

2. Even Phase

Compare and swap elements at even indices with their next neighbors (i = 0, 2, 4,...)

3. Repeat

Continue alternating between odd and even phases until no swaps occur in a full cycle

4. Completion

When a complete odd-even cycle with no swaps occurs, the array is sorted

Pseudocode

procedure oddEvenSort(A: list of sortable items)
    n = length(A)
    sorted = false
    
    while not sorted
        sorted = true
        
        // Odd phase (i = 1, 3, 5,...)
        for i = 1 to n-2 step 2
            if A[i] > A[i+1] then
                swap(A[i], A[i+1])
                sorted = false
        
        // Even phase (i = 0, 2, 4,...)
        for i = 0 to n-2 step 2
            if A[i] > A[i+1] then
                swap(A[i], A[i+1])
                sorted = false

Implementation

Below is a Python implementation of Odd-Even Sort. The algorithm alternates between odd and even phases until no swaps occur in a full cycle.

Python Implementation 
def odd_even_sort(arr):
    n = len(arr)
    sorted = False
    
    while not sorted:
        sorted = True
        
        # Odd phase (i = 1, 3, 5, ...)
        for i in range(1, n-1, 2):
            if arr[i] > arr[i+1]:
                arr[i], arr[i+1] = arr[i+1], arr[i]
                sorted = False
                
        # Even phase (i = 0, 2, 4, ...)
        for i in range(0, n-1, 2):
            if arr[i] > arr[i+1]:
                arr[i], arr[i+1] = arr[i+1], arr[i]
                sorted = False
                
    return arr

# Example usage
arr = [38, 27, 43, 3, 9, 82, 10, 15]
sorted_arr = odd_even_sort(arr)
print("Sorted array:", sorted_arr)
# Output: Sorted array: [3, 9, 10, 15, 27, 38, 43, 82]

Applications

Odd-Even Sort is particularly useful in specific scenarios:

  • Parallel computing environments
  • Educational purposes to demonstrate parallel sorting
  • Hardware description languages (VHDL/Verilog)
  • Sorting small to medium datasets
  • As an alternative to bubble sort with parallel potential

Parallelism Advantage

The main advantage of Odd-Even Sort is its parallelization capability:

  • All comparisons in the odd phase can be performed simultaneously
  • All comparisons in the even phase can be performed simultaneously
  • This allows for O(n) time complexity in parallel implementations
  • Particularly efficient on SIMD parallel computing architectures
