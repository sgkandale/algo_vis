Quick Sort

Sorting Algorithm
Time Complexity: O(n log n)

Algorithm Overview

Quick Sort is an efficient, comparison-based, divide and conquer sorting algorithm. It works by selecting a 'pivot' element from the array and partitioning the other elements into two sub-arrays according to whether they are less than or greater than the pivot. The sub-arrays are then sorted recursively.

Quick Sort is one of the most efficient sorting algorithms and is often the best choice for large datasets. Its average-case time complexity is O(n log n), though it can degrade to O(n²) in the worst case if the pivot selection is poor.

Performance

Average O(n log n), worst O(n²) but rare with good pivot selection

Space

O(log n) stack space for recursion

Approach

Divide and conquer with in-place partitioning

Advantage

Often faster than other O(n log n) algorithms in practice

Algorithm Steps

Quick Sort involves the following steps to sort an array:

1. Choose Pivot

Select a pivot element from the array (commonly the last element)

2. Partition

Rearrange elements so values less than pivot come before, greater come after

3. Pivot Position

Place the pivot in its correct sorted position

4. Recursive Sort

Recursively apply to the sub-arrays before and after the pivot

Pseudocode

function quickSort(array, low, high)
    if low < high
        // Partition the array
        pivotIndex = partition(array, low, high)
        
        // Recursively sort elements before and after partition
        quickSort(array, low, pivotIndex - 1)
        quickSort(array, pivotIndex + 1, high)

function partition(array, low, high)
    // Choose pivot (here we use last element)
    pivot = array[high]
    i = low - 1
    
    for j from low to high - 1
        if array[j] <= pivot
            i = i + 1
            swap array[i] and array[j]
    
    // Place pivot in correct position
    swap array[i+1] and array[high]
    return i + 1

Implementation

Below is a Python implementation of the Quick Sort algorithm. The algorithm consists of the main quick_sort() function and a partition() function that rearranges elements around the pivot.

Python Implementation 
def quick_sort(arr, low, high):
    if low < high:
        # Partition the array
        pivot_index = partition(arr, low, high)
        
        # Recursively sort elements before and after partition
        quick_sort(arr, low, pivot_index - 1)
        quick_sort(arr, pivot_index + 1, high)

def partition(arr, low, high):
    # Choose pivot (last element)
    pivot = arr[high]
    i = low - 1
    
    for j in range(low, high):
        if arr[j] <= pivot:
            i += 1
            # Swap elements
            arr[i], arr[j] = arr[j], arr[i]
    
    # Place pivot in correct position
    arr[i+1], arr[high] = arr[high], arr[i+1]
    return i + 1

# Example usage
arr = [10, 7, 8, 9, 1, 5]
quick_sort(arr, 0, len(arr)-1)
print("Sorted array:", arr)

Applications

Quick Sort is used in various scenarios where efficient sorting is required:

  • Standard library sort implementations (like in C, Java, etc.)
  • Sorting large datasets in memory
  • When average-case performance matters more than worst-case
  • Implementations where cache performance is important
  • Parallel processing environments

Complexity Analysis

Quick Sort's performance depends on pivot selection:

Best Case: O(n log n)

When pivot divides array into equal parts

Average Case: O(n log n)

With random pivot selection

Worst Case: O(n²)

When pivot is smallest or largest element

Pivot Selection

Different pivot selection strategies affect performance:

First/Last

Simple but worst-case O(n²) for sorted arrays

Random

Reduces chance of worst-case to negligible

Median-of-Three

Takes median of first, middle, last elements
