Radix Sort

Non-Comparison Sorting Algorithm
Time Complexity: O(d*(n+b))

Algorithm Overview

Radix Sort is a non-comparative integer sorting algorithm that sorts numbers by processing individual digits. It works by distributing numbers into buckets based on each digit, starting from the least significant digit (LSD) to the most significant digit (MSD). Radix Sort can be implemented using a stable sort (like Counting Sort) as its subroutine.

Unlike comparison-based algorithms, Radix Sort takes advantage of the integer structure to achieve linear time complexity in many practical scenarios. It's particularly efficient for sorting large collections of numbers with a fixed digit length.

Key Feature

Digit-by-digit sorting without direct comparisons

Performance

O(d*(n+b)) complexity (d=digits, n=elements, b=base)

Space

Requires O(n + b) additional space

Approach

Distribute numbers into buckets based on each digit position

Algorithm Steps

Radix Sort processes numbers digit by digit, from the least significant to the most significant:

1. Find Maximum

Determine the maximum number to know how many digits to process.

2. Initialize Buckets

Create 10 buckets (0-9) for base-10 digits.

3. Process Digits

For each digit position (starting from LSD):

4. Distribute to Buckets

Place each number in the bucket corresponding to its current digit.

5. Collect from Buckets

Gather numbers from buckets in order to form a new list.

6. Repeat

Repeat the process for the next digit until all digits are processed.

Pseudocode

function radixSort(arr):
    // Find the maximum number to know number of digits
    max_num = max(arr)
    exp = 1  // Start with least significant digit
    
    while max_num / exp > 0:
        // Create 10 buckets (0-9)
        buckets = array of 10 empty lists
        
        // Place numbers in buckets based on current digit
        for i from 0 to n-1:
            digit = (arr[i] / exp) % 10
            buckets[digit].append(arr[i])
        
        // Collect numbers from buckets in order
        index = 0
        for bucket_num from 0 to 9:
            for each num in buckets[bucket_num]:
                arr[index] = num
                index += 1
                
        // Move to next significant digit
        exp *= 10

Implementation

Below is a Python implementation of Radix Sort using base-10 digits. The algorithm uses buckets to distribute numbers based on each digit position.

Python Implementation 
def radix_sort(arr):
    # Find the maximum number to know the number of digits
    max_num = max(arr)
    
    # Start with the least significant digit
    exp = 1
    
    while max_num // exp > 0:
        # Create 10 buckets for digits 0-9
        buckets = [[] for _ in range(10)]
        
        # Place numbers in buckets based on current digit
        for num in arr:
            digit = (num // exp) % 10
            buckets[digit].append(num)
            
        # Collect numbers from buckets in order
        index = 0
        for i in range(10):
            for num in buckets[i]:
                arr[index] = num
                index += 1
                
        # Move to next significant digit
        exp *= 10

# Example usage
arr = [170, 45, 75, 90, 802, 24, 2, 66]
radix_sort(arr)
print("Sorted array:", arr)  # Output: [2, 24, 45, 66, 75, 90, 170, 802]

Applications

Radix Sort is particularly effective for specific types of data:

  • Sorting large collections of integers
  • Sorting fixed-length strings (e.g., phone numbers)
  • Applications requiring stable integer sorting
  • Computer graphics and numerical computations
  • Card sorting machines and punch card systems

Advantages & Disadvantages

Advantages

  • Linear time complexity for fixed-digit numbers
  • Stable sorting algorithm
  • Efficient for large datasets with limited digit length
  • No comparisons between elements

Disadvantages

  • Requires additional memory for buckets
  • Only works for integer and fixed-length string data
  • Inefficient for numbers with many digits
  • Performance depends on the base used

Complexity Analysis

Radix Sort's performance depends on the number of digits (d) and the base (b):

Time Complexity: O(d*(n + b))
Space Complexity: O(n + b)

Where n is the number of elements, d is the number of digits, and b is the base (number of buckets). When d is constant and b is O(n), Radix Sort runs in linear time O(n).
