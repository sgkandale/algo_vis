Smoothsort

Sorting Algorithm
Time Complexity: O(n log n)

Algorithm Overview

Smoothsort is an adaptive variant of heapsort developed by Edsger Dijkstra. It has a time complexity of O(n log n) in the worst case, and O(n) in the best case (when the input is already sorted). It is an in-place, unstable sorting algorithm that uses a Leonardo heap - a heap-like data structure composed of Leonardo trees.

The algorithm is particularly efficient for partially sorted data and minimizes the number of comparisons and swaps. Its adaptive nature makes it suitable for scenarios where data is already partially ordered.

Key Feature

Adaptive algorithm with O(n) best-case performance

Performance

O(n log n) worst-case, O(n) best-case (when input is sorted)

Space

O(1) additional space - in-place sorting algorithm

Approach

Leonardo heap construction and deconstruction

Algorithm Steps

Smoothsort processes the array using Leonardo heaps:

1. Leonardo Heaps

The algorithm uses Leonardo numbers (L(n) = L(n-1) + L(n-2) + 1) to form a heap structure

2. Heap Construction

Build the heap by adding elements while maintaining heap properties

3. Heap Ordering

Ensure each Leonardo tree satisfies the heap property (parent > children)

4. Heap Balancing

Maintain the heap by balancing tree sizes according to Leonardo numbers

5. Heap Deconstruction

Remove the root (largest element) and place it in sorted position

Pseudocode

function smoothSort(arr):
    n = length(arr)
    p = n - 1  // Last element index
    q = p
    r = 0
    
    // Precompute Leonardo numbers
    leo = generateLeonardoNumbers(n)
    
    // Build the heap
    for i from 0 to n-1:
        if p % 2 == 1:
            p = (p - 1) / 2
            q = q - leo[p]
        else:
            p = p + 1
            q = leo[p] - 1
            r = q
        
        // Restore heap property
        siftDown(arr, r, p)
    
    // Deconstruct the heap
    for i from n-1 down to 1:
        // Swap root with last element
        swap(arr[0], arr[i])
        
        // Reduce heap size
        p = p - 1
        if p < 0:
            q = q - 1
            p = indexOf(q)
        
        // Restore heap property
        siftDown(arr, 0, p)

function siftDown(arr, root, treeSize):
    current = root
    while treeSize > 1:
        left = current - (treeSize - leo[treeSize-2])
        right = current - 1
        
        if arr[current] >= arr[left] and arr[current] >= arr[right]:
            break
        
        if arr[left] > arr[right]:
            swap(arr[current], arr[left])
            current = left
            treeSize = treeSize - 1
        else:
            swap(arr[current], arr[right])
            current = right
            treeSize = treeSize - 2

Implementation

Below is a Python implementation of Smoothsort. The algorithm uses Leonardo numbers to form heap structures and efficiently sorts data:

Python Implementation 
def leonardo_numbers(n):
    # Generate Leonardo numbers up to n
    a, b = 1, 1
    numbers = []
    while a < n:
        numbers.append(a)
        a, b = b, a + b + 1
    return numbers

def smooth_sort(arr):
    n = len(arr)
    if n <= 1:
        return arr
        
    leo = leonardo_numbers(n)
    
    # Build the heap
    heap = []
    for i in range(n):
        if len(heap) >= 2 and heap[-2] == heap[-1] + 1:
            heap.pop()
            heap[-1] += 1
        else:
            if len(heap) >= 1 and heap[-1] == 1:
                heap.append(0)
            else:
                heap.append(1)
        
        # Restore heap property
        sift_down(arr, i, heap, leo)
    
    # Deconstruct the heap
    for i in range(n-1, 0, -1):
        if heap[-1] < 2:
            heap.pop()
        else:
            k = heap.pop()
            left, right = get_child_trees(i, k, leo)
            heap.append(right)
            sift_down(arr, left, heap, leo)
            heap.append(left)
            sift_down(arr, i, heap, leo)
    
    return arr

def sift_down(arr, root, heap, leo):
    k = heap[-1]
    if k < 2:
        return
    
    r = root
    while k >= 2:
        right = r - 1
        left = r - 1 - leo[k-2]
        
        if arr[r] >= arr[left] and arr[r] >= arr[right]:
            break
            
        if arr[left] > arr[right]:
            arr[r], arr[left] = arr[left], arr[r]
            r = left
            k = k - 1
        else:
            arr[r], arr[right] = arr[right], arr[r]
            r = right
            k = k - 2

# Example usage
arr = [3, 1, 5, 4, 2, 9, 7, 8, 6]
sorted_arr = smooth_sort(arr.copy())
print("Original array:", arr)
print("Sorted array:", sorted_arr)

Applications

Smoothsort is particularly useful in specialized scenarios:

  • Systems requiring adaptive sorting algorithms
  • Real-time systems with partially sorted inputs
  • Memory-constrained environments (in-place sorting)
  • Applications where data is frequently nearly sorted
  • Educational demonstrations of adaptive heapsort

Leonardo Numbers

Leonardo numbers (similar to Fibonacci numbers) are used to determine heap sizes:

L(0) = 1, L(1) = 1

L(n) = L(n-1) + L(n-2) + 1 for n > 1
